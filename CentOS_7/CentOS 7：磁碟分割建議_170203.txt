在 Linux 下進行手動分割的各個分割區大小和掛載點：


  在進行 Linux 的磁碟分割之前，有二點要注意的是：
  
  
  1.一顆硬碟最多只能分割成四個主分割區。
  
  2.有一個特殊分割區格式 (BIOS Boot) 以及七個重要的目錄需要配置磁碟空間，說明如下：

    BIOS boot	: 它是一個特殊的分割區格式，用來讓不支援 UEFI 的主機板，GRUB Boot Loader 可以由 GPT 格式的分割區開機。
				  
    /			: 
	/boot		:
	/home		:
	/tmp		:
	/usr		:
	/var		:
	swap		: 

  
  2.BIOS Boot 需要一個獨立的分割區，大小約 1MB 就足夠了。
	
  3.Linux 有七個重要的目錄，分別是 '/', '/boot', '/usr', '/var', '/tmp', '/home' 和 swap，
  
    這七個目錄最好有獨立的分割區。
	
	
  因此，為了滿足上述的需求。
  建議將主硬碟分割成四個主分割區，分別配置給 BIOS Boot, '/boot', '/usr' 和 '/'。
  其中，BIOS Boot, '/boot' 和 '/usr' 的裝置類型設定為標準分割區，
  而 '/' 的裝置類型設定為 LVM，並修改其 Volumn Group 的空間容量。
 
  底下將逐一說明各分割區的功能和配置的空間大小：(以 40G 磁碟空間為例)
  

  BIOS Boot
  
    這個分割區我配置了 2MB 的磁碟空間，這是參考鳥哥的 Linux 私房菜的設定。
	檔案系統的選項為 BIOS Boot。
	我暫時還不知道為什麼。
	

  /boot

    這個目錄是用來存放啟動 Linux 核心所需的檔案，一般來說只需配置 200MB 即可，
    配置過多的硬碟空間反而會造成硬碟空間的浪費。
    我配置了 500MB 的磁碟空間，檔案系統為 ext4。

  
  /usr

    這個目錄儲存了幾乎所有 Linux 系統中的應用軟體套件、說明文件和函數類別庫。
	所以隨著 Linux 系統的使用時間愈久，安裝的應用軟體愈多，需要的磁碟空間也會愈來愈大。
	一般來說，配置 5~10GB 給這個目錄應該就足夠使用了。
	我配置了 5GB 的磁碟空間，檔案系統為 ext4。
	
	
  /

    根目錄，Linux 檔案系統中的所有目錄都在此目錄之下，所以配置了 10GB 的磁碟空間給它。
	由於主硬碟有最多只能分割成四個主分割區﹝primary partition﹞的限制，
	而且已經用掉三個主分割區了，所以此分割區的裝置類型必須設定為 LVM，
	並修改其 Volume Group 的大小政策為 '固定'，以利後續分割其他的目錄使用。
	在此，我將 Volume Group 的磁碟空間設定為 30G，檔案系統為 xfs。
	
	
  swap

    置換空間，也就是虛擬記憶體空間。一般來說，這個分割區的大小是電腦記憶體的 2倍。
    舉例來說，如果電腦的記憶體是 1GB，那麼最少要分割 2GB 的空間給 swap。
    我配置了 2GB。
  

  /var

    這個目錄存放各種關於 Linux kernel 和應用軟體的檔案，如日誌檔和訊息檔。
	一般來說，只要配置 2~5GB 就足夠了。
	我配置了 2GB。

  
  /tmp

    此目錄用來存放暫存檔，在每次 Linux 系統重新開機時就會全部清空，
    在燒錄光碟時，此目錄也會用來暫存燒錄光碟所需的ISO檔。
	因此，這個分割區適合的大小是 2~5GB。
	考慮到我不會用 Linux 系統來燒錄光碟，所以我只配置了 500MB。

  /home

    家目錄，所有個人檔案都存放於此，包括文件檔、音樂檔、影片檔等`.
	所以建議將剩餘的空間就全部都配置給此目錄。
	
	
  這樣的硬碟分割方式，不僅可以讓 Linux 系統更有效率地使用硬碟，
  當其中某些目錄毀損了，只需針對 /、/boot、/usr 這些目錄的分割區重新安裝，
  就可以讓系統恢復正常了。
  若損壞很嚴重且無法藉由還原部分目錄讓 Linux 系統恢復正常時，就只能重灌 Linux 了。
